# Requirements Document

## Introduction

This feature transforms the existing aggregate performance page into a comprehensive closed markets viewer that enables users to verify AI recommendation accuracy and value through detailed per-market performance analysis. The system will display resolved markets using existing UI patterns and provide sophisticated performance tracking including profit/loss simulations, recommendation accuracy metrics, and risk-adjusted return analysis.

## Glossary

- **Closed_Market**: A prediction market on Polymarket that has been resolved with a final outcome
- **AI_Recommendation**: A trading signal generated by the TradeWizard multi-agent system containing entry/exit zones, confidence levels, and risk assessment
- **Performance_Tab**: A new UI component within market detail views that displays AI recommendation performance metrics
- **Recommendation_History**: The complete timeline of AI recommendations issued for a specific market
- **Simulated_Portfolio**: A hypothetical portfolio that tracks profit/loss as if a user had followed AI recommendations with a specified investment amount
- **Entry_Point**: The price level at which an AI recommendation suggests entering a position
- **Exit_Point**: The price level at which an AI recommendation suggests closing a position
- **ROI_Metric**: Return on Investment calculation showing percentage gain/loss from following recommendations
- **Confidence_Level**: The AI system's probability estimate for a market outcome
- **Market_Detail_View**: The existing UI component that displays comprehensive information about a single market
- **Homepage_UI_Pattern**: The existing card-based market display interface used on the TradeWizard homepage
- **Supabase_Database**: The PostgreSQL database managed by Supabase containing market and recommendation data
- **Risk_Adjusted_Return**: Performance metric that accounts for the volatility and risk taken to achieve returns

## Requirements

### Requirement 1: Display Closed Markets

**User Story:** As a user, I want to view all closed markets in a familiar interface, so that I can easily browse resolved markets and assess AI performance.

#### Acceptance Criteria

1. THE Performance_Page SHALL retrieve all Closed_Markets from the Supabase_Database
2. THE Performance_Page SHALL display Closed_Markets using the Homepage_UI_Pattern
3. THE Performance_Page SHALL sort Closed_Markets by resolution date in descending order
4. WHEN a Closed_Market has no AI_Recommendations, THE Performance_Page SHALL display the market with a visual indicator showing no analysis was performed
5. THE Performance_Page SHALL display market resolution outcome alongside market title
6. THE Performance_Page SHALL paginate results when more than 20 Closed_Markets exist

### Requirement 2: Market Interaction Consistency

**User Story:** As a user, I want closed markets to behave like homepage markets, so that I have a consistent experience across the application.

#### Acceptance Criteria

1. WHEN a user clicks on a Closed_Market, THE Performance_Page SHALL navigate to the Market_Detail_View
2. THE Market_Detail_View SHALL display all standard market information for Closed_Markets
3. THE Market_Detail_View SHALL disable trading actions for Closed_Markets
4. THE Market_Detail_View SHALL display the final resolution outcome prominently

### Requirement 3: Performance Tab Integration

**User Story:** As a user, I want to see a performance analysis tab in market details, so that I can evaluate how AI recommendations performed for that specific market.

#### Acceptance Criteria

1. WHEN viewing a Closed_Market with AI_Recommendations, THE Market_Detail_View SHALL display a Performance_Tab
2. WHEN viewing a Closed_Market without AI_Recommendations, THE Market_Detail_View SHALL NOT display a Performance_Tab
3. WHEN a user clicks the Performance_Tab, THE Market_Detail_View SHALL display performance metrics without page navigation
4. THE Performance_Tab SHALL load performance data from the Supabase_Database

### Requirement 4: Profit/Loss Simulation

**User Story:** As a user, I want to simulate profit/loss from following AI recommendations, so that I can understand the financial value of the AI system.

#### Acceptance Criteria

1. THE Performance_Tab SHALL provide an input field for specifying simulated investment amount
2. WHEN a user enters an investment amount, THE Simulated_Portfolio SHALL calculate total profit/loss across all AI_Recommendations for that market
3. THE Simulated_Portfolio SHALL calculate profit/loss for each individual AI_Recommendation
4. WHEN an AI_Recommendation includes an Entry_Point and Exit_Point, THE Simulated_Portfolio SHALL calculate returns based on those price levels
5. WHEN an AI_Recommendation was not followed to completion before market closure, THE Simulated_Portfolio SHALL calculate returns based on final market resolution price
6. THE Simulated_Portfolio SHALL display cumulative profit/loss over time as a line chart
7. THE Simulated_Portfolio SHALL account for Polymarket trading fees in profit/loss calculations

### Requirement 5: Recommendation Accuracy Tracking

**User Story:** As a user, I want to see how accurate AI recommendations were, so that I can assess the reliability of the AI system.

#### Acceptance Criteria

1. THE Performance_Tab SHALL calculate accuracy percentage for all AI_Recommendations
2. WHEN an AI_Recommendation predicted the correct market outcome, THE Performance_Tab SHALL count it as accurate
3. THE Performance_Tab SHALL display the number of accurate recommendations versus total recommendations
4. THE Performance_Tab SHALL display average Confidence_Level across all AI_Recommendations
5. THE Performance_Tab SHALL display a correlation metric between Confidence_Level and actual accuracy

### Requirement 6: Entry/Exit Point Visualization

**User Story:** As a user, I want to see recommended entry/exit points overlaid on actual market movements, so that I can evaluate timing accuracy.

#### Acceptance Criteria

1. THE Performance_Tab SHALL display a price chart showing market price over time
2. THE Performance_Tab SHALL overlay Entry_Point markers on the price chart at the timestamp when recommendations were issued
3. THE Performance_Tab SHALL overlay Exit_Point markers on the price chart at the timestamp when exit signals were issued
4. WHEN a user hovers over an Entry_Point or Exit_Point marker, THE Performance_Tab SHALL display the full AI_Recommendation details in a tooltip
5. THE Performance_Tab SHALL use distinct visual styling for profitable versus unprofitable recommendation periods

### Requirement 7: ROI Metrics Display

**User Story:** As a user, I want to see return on investment metrics, so that I can quantify the financial performance of AI recommendations.

#### Acceptance Criteria

1. THE Performance_Tab SHALL calculate and display total ROI_Metric as a percentage
2. THE Performance_Tab SHALL calculate and display average ROI_Metric per recommendation
3. THE Performance_Tab SHALL calculate and display best performing recommendation ROI_Metric
4. THE Performance_Tab SHALL calculate and display worst performing recommendation ROI_Metric
5. THE Performance_Tab SHALL display ROI_Metric for each individual AI_Recommendation in the Recommendation_History

### Requirement 8: Recommendation Timeline

**User Story:** As a user, I want to see the complete timeline of AI recommendations, so that I can understand how the AI's analysis evolved over the market's lifetime.

#### Acceptance Criteria

1. THE Performance_Tab SHALL display the Recommendation_History in chronological order
2. THE Recommendation_History SHALL display the timestamp for each AI_Recommendation
3. THE Recommendation_History SHALL display the recommendation type (buy, sell, hold) for each AI_Recommendation
4. THE Recommendation_History SHALL display the Confidence_Level for each AI_Recommendation
5. THE Recommendation_History SHALL display the market price at the time each AI_Recommendation was issued
6. WHEN a user clicks on a recommendation in the Recommendation_History, THE Performance_Tab SHALL highlight the corresponding period on the price chart

### Requirement 9: Risk-Adjusted Returns Analysis

**User Story:** As a user, I want to see risk-adjusted performance metrics, so that I can evaluate whether returns justified the risk taken.

#### Acceptance Criteria

1. THE Performance_Tab SHALL calculate and display a Sharpe ratio for the Simulated_Portfolio
2. THE Performance_Tab SHALL calculate and display maximum drawdown percentage
3. THE Performance_Tab SHALL calculate and display volatility of returns
4. THE Performance_Tab SHALL display Risk_Adjusted_Return metrics alongside raw ROI_Metric
5. THE Performance_Tab SHALL provide explanatory tooltips for each risk metric

### Requirement 10: Confidence Level vs Outcome Analysis

**User Story:** As a user, I want to see how AI confidence levels correlated with actual outcomes, so that I can calibrate my trust in future confidence scores.

#### Acceptance Criteria

1. THE Performance_Tab SHALL display a scatter plot with Confidence_Level on x-axis and actual outcome on y-axis
2. THE Performance_Tab SHALL calculate and display calibration error for Confidence_Levels
3. WHEN Confidence_Level was high and outcome was correct, THE Performance_Tab SHALL highlight this as strong calibration
4. WHEN Confidence_Level was high and outcome was incorrect, THE Performance_Tab SHALL highlight this as poor calibration
5. THE Performance_Tab SHALL display average Confidence_Level for accurate versus inaccurate recommendations

### Requirement 11: Data Persistence and Retrieval

**User Story:** As a developer, I want efficient data retrieval from Supabase, so that the performance page loads quickly even with many closed markets.

#### Acceptance Criteria

1. THE Performance_Page SHALL query only Closed_Markets from the Supabase_Database
2. THE Performance_Page SHALL use database indexes for efficient filtering by market status
3. THE Performance_Tab SHALL query AI_Recommendations only for the specific market being viewed
4. THE Performance_Tab SHALL cache performance calculations to avoid redundant computation
5. WHEN performance data is not available, THE Performance_Tab SHALL display a loading state

### Requirement 12: Performance Comparison Baseline

**User Story:** As a user, I want to compare AI performance against baseline strategies, so that I can understand the AI's added value.

#### Acceptance Criteria

1. THE Performance_Tab SHALL calculate profit/loss for a "buy and hold from first recommendation" baseline strategy
2. THE Performance_Tab SHALL calculate profit/loss for a "random entry/exit" baseline strategy
3. THE Performance_Tab SHALL display AI performance versus baseline strategies side-by-side
4. THE Performance_Tab SHALL highlight when AI performance exceeds baseline strategies
5. THE Performance_Tab SHALL calculate statistical significance of AI outperformance

### Requirement 13: Export Performance Data

**User Story:** As a user, I want to export performance data, so that I can conduct my own analysis or share results.

#### Acceptance Criteria

1. THE Performance_Tab SHALL provide an export button
2. WHEN a user clicks the export button, THE Performance_Tab SHALL generate a CSV file containing all performance metrics
3. THE CSV file SHALL include timestamps, recommendations, prices, and calculated metrics
4. THE CSV file SHALL download to the user's device
5. THE CSV file SHALL be named with the market title and export timestamp

### Requirement 14: Mobile Responsive Performance View

**User Story:** As a mobile user, I want to view performance metrics on my phone, so that I can assess AI performance on any device.

#### Acceptance Criteria

1. THE Performance_Tab SHALL display all metrics in a mobile-optimized layout
2. THE Performance_Tab SHALL stack charts vertically on mobile devices
3. THE Performance_Tab SHALL use touch-friendly controls for chart interactions
4. THE Performance_Tab SHALL maintain readability of all text and numbers on small screens
5. WHEN screen width is below 768 pixels, THE Performance_Tab SHALL use mobile layout

### Requirement 15: Error Handling and Edge Cases

**User Story:** As a user, I want clear feedback when performance data is incomplete, so that I understand limitations in the analysis.

#### Acceptance Criteria

1. WHEN a Closed_Market has incomplete price history, THE Performance_Tab SHALL display a warning message
2. WHEN an AI_Recommendation is missing Entry_Point or Exit_Point data, THE Performance_Tab SHALL exclude it from profit/loss calculations and display a notice
3. WHEN the Supabase_Database query fails, THE Performance_Page SHALL display an error message and retry option
4. WHEN no Closed_Markets exist, THE Performance_Page SHALL display an empty state message
5. IF performance calculations encounter division by zero, THEN THE Performance_Tab SHALL handle the error gracefully and display "N/A" for that metric
